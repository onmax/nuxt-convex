title: File Storage
description: Enable Convex storage scaffolding in nuxt-convex and wire the generated helpers into your app code.
---

Set `convex.storage` to `true` when you want the Nuxt module to scaffold the basic Convex storage functions for you. The module also enables the storage-specific alias and auto-imports for the shared storage composables.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-convex'],
  convex: {
    storage: true,
  },
})
```

## What the module generates

When storage is enabled, the module writes `convex/_hub/storage.ts` if the file does not exist yet.

```ts [convex/_hub/storage.ts]
import { v } from 'convex/values'
import { mutation, query } from '../_generated/server'

export const generateUploadUrl = mutation(async (ctx) => {
  return await ctx.storage.generateUploadUrl()
})

export const getUrl = query({
  args: { storageId: v.id('_storage') },
  handler: async (ctx, { storageId }) => {
    return await ctx.storage.getUrl(storageId)
  },
})

export const remove = mutation({
  args: { storageId: v.id('_storage') },
  handler: async (ctx, { storageId }) => {
    await ctx.storage.delete(storageId)
  },
})
```

::note
The generated file is a starting point. You can extend it with your own metadata mutation or authorization rules after the scaffold exists.
::

## What becomes available in Nuxt

With `storage: true`, the module adds:

- the `#convex/storage` alias
- `useConvexStorage` and `useConvexUpload` auto-imports from `#convex/storage`
- the scaffolded `convex/_hub/storage.ts` file when it does not exist yet

## Persist metadata separately

The scaffold only handles upload URLs, public URLs, and deletion. If you want to track filenames, MIME types, or ownership, add your own mutation.

```ts [convex/files.ts]
import { v } from 'convex/values'
import { mutation } from './_generated/server'

export const saveUpload = mutation({
  args: {
    storageId: v.id('_storage'),
    name: v.string(),
    type: v.string(),
  },
  handler: async (ctx, args) => {
    return await ctx.db.insert('uploads', {
      ...args,
      createdAt: Date.now(),
    })
  },
})
```

## Frontend flow

The upload flow stays close to the shared Vue API. In Nuxt, import the generated API from `#convex/api` and the storage composable from `#convex/storage`.

```vue [app/components/FileUpload.vue]
<script setup lang="ts">
import { api } from '#convex/api'
import { useConvexUpload } from '#convex/storage'

const { mutate: saveUpload } = useConvexMutation(api.files.saveUpload)
const { upload, isUploading, error } = useConvexUpload({
  onSuccess: async (storageId, file) => {
    await saveUpload({
      storageId,
      name: file.name,
      type: file.type,
    })
  },
})

async function onSelect(event: Event) {
  const file = (event.target as HTMLInputElement).files?.[0]
  if (file)
    await upload(file)
}
</script>
```

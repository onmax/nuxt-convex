Use the Convex R2 component when you want bucket-backed uploads instead of Convex storage objects. The canonical validation flow lives in the playground dashboard under `Cloudflare R2`.

## Install the dependencies

::code-group{sync="pm"}

```bash [pnpm]
pnpm add @convex-dev/r2
```

```bash [npm]
npm install @convex-dev/r2
```

```bash [yarn]
yarn add @convex-dev/r2
```

```bash [bun]
bun add @convex-dev/r2
```

::

## Enable Nuxt support

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-convex'],
  convex: {
    r2: true,
  },
})
```

This switch adds the `useConvexR2Upload` auto-import. It does not add a new virtual module or a new `#convex/*` alias.

## Register the Convex component

```ts [convex/convex.config.ts]
import r2 from '@convex-dev/r2/convex.config.js'
import { defineApp } from 'convex/server'

const app = defineApp()
app.use(r2)

export default app
```

## Create the client API

```ts [convex/r2.ts]
import type { DataModel } from './_generated/dataModel'
import { R2 } from '@convex-dev/r2'
import { componentsGeneric } from 'convex/server'

const components = componentsGeneric()

export const r2 = new R2(components.r2)

export const {
  generateUploadUrl,
  syncMetadata,
  listMetadata,
  deleteObject,
} = r2.clientApi<DataModel>()
```

The playground exercises upload, metadata listing, and deletion through this same client API.

## Add the required environment

Set the R2 credentials in the Convex environment before you test uploads.

```bash [.env]
R2_BUCKET=your-bucket
R2_ENDPOINT=https://<account-id>.r2.cloudflarestorage.com
R2_ACCESS_KEY_ID=replace-me
R2_SECRET_ACCESS_KEY=replace-me
```

## Use the Nuxt helpers

```vue [app/components/R2Upload.vue]
<script setup lang="ts">
import { api } from '#convex/api'

const { upload, progress } = useConvexR2Upload(api.r2)
</script>
```

`useConvexR2Upload` is the supported Nuxt-side entrypoint for R2 uploads. It requests a signed upload URL from Convex, uploads the file, and syncs metadata back through the generated client API.

::note
Use the `Cloudflare R2` section in the playground dashboard as the validation reference. It covers upload, list, open, and delete against the same API surface that this page documents.
::

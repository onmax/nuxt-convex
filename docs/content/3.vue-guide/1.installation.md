## Install the packages

::code-group{sync="pm"}

```bash [pnpm]
pnpm add @onmax/convex-vue convex
```

```bash [npm]
npm install @onmax/convex-vue convex
```

```bash [yarn]
yarn add @onmax/convex-vue convex
```

::

## Register the plugin

```ts [src/main.ts]
import { convexVue } from '@onmax/convex-vue'
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

app.use(convexVue, {
  url: import.meta.env.VITE_CONVEX_URL,
})

app.mount('#app')
```

::tip
Store your Convex deployment URL in a `VITE_CONVEX_URL` environment variable. Vite exposes it through `import.meta.env`.
::

## Plugin options

::field-group
  ::field{name="url" type="string"}
  The Convex deployment URL. When omitted, the plugin starts disconnected. Use the [controller](/vue-guide/controller) to connect later.
  ::

  ::field{name="clientOptions" type="ConvexClientOptions"}
  Options forwarded to the underlying `ConvexClient` and `ConvexHttpClient` constructors.
  ::

  ::field{name="server" type="boolean"}
  Default `true`. Controls whether `useConvexQuery` fetches data on the server with `ConvexHttpClient` before the client subscribes.
  ::
::

## Generate the typed API

Run the Convex development server so the generated files exist before you write frontend code.

```bash [Terminal]
npx convex dev
```

The command creates `convex/_generated/api` and `convex/_generated/server`. Every example in this documentation imports from these generated modules.

## Verify the setup

```vue [src/components/TaskList.vue]
<script setup lang="ts">
import { useConvexQuery } from '@onmax/convex-vue'
import { api } from '../convex/_generated/api'

const { data, suspense } = useConvexQuery(api.tasks.list, {})
await suspense()
</script>

<template>
  <ul>
    <li v-for="task in data" :key="task._id">
      {{ task.text }}
    </li>
  </ul>
</template>
```

If the list renders your Convex data, the setup is complete.

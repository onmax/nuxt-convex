## Mutations

`useConvexMutation` wraps a Convex mutation function. Call `mutate()` to execute it and track loading and error state reactively.

```vue [src/components/AddTask.vue]
<script setup lang="ts">
import { useConvexMutation } from '@onmax/convex-vue'
import { api } from '../convex/_generated/api'

const { mutate, isPending, error } = useConvexMutation(api.tasks.create)

async function onSubmit(text: string) {
  await mutate({ text, completed: false })
}
</script>

<template>
  <form @submit.prevent="onSubmit('Buy groceries')">
    <button :disabled="isPending">
      Add task
    </button>
    <p v-if="error">
      {{ error.message }}
    </p>
  </form>
</template>
```

### Return values

::field-group
::field{name="mutate" type="(args) => Promise<T>"}
Execute the mutation. Returns the mutation result.
::

::field{name="error" type="Ref<Error | null>"}
The latest mutation error. Resets to `null` on the next call.
::

::field{name="isPending" type="ComputedRef<boolean>"}
`true` while any call is in flight. Supports concurrent calls.
::
::

### Optimistic updates

Pass an `optimisticUpdate` function to update the local cache before the server confirms.

```ts [src/components/ToggleTask.vue]
import { useConvexMutation } from '@onmax/convex-vue'
import { api } from '../convex/_generated/api'

const { mutate } = useConvexMutation(api.tasks.toggle, {
  optimisticUpdate(localStore, args) {
    const tasks = localStore.getQuery(api.tasks.list, {})
    if (!tasks)
      return
    localStore.setQuery(
      api.tasks.list,
      {},
      tasks.map(t => t._id === args.id ? { ...t, completed: !t.completed } : t),
    )
  },
})
```

The local store reverts automatically if the server rejects the mutation.

## Actions

`useConvexAction` wraps a Convex action. Actions can call external APIs and run longer logic that does not fit in a transaction.

```vue [src/components/SendEmail.vue]
<script setup lang="ts">
import { useConvexAction } from '@onmax/convex-vue'
import { api } from '../convex/_generated/api'

const { execute, isPending, error } = useConvexAction(api.emails.send)

async function onClick() {
  await execute({ to: 'user@example.com', subject: 'Hello' })
}
</script>
```

### Return values

::field-group
::field{name="execute" type="(args) => Promise<T>"}
Run the action. Returns the action result.
::

::field{name="error" type="Ref<Error | null>"}
The latest action error.
::

::field{name="isPending" type="ComputedRef<boolean>"}
`true` while any call is in flight.
::
::

## When to use which

|                        | Mutation        | Action                               |
| ---------------------- | --------------- | ------------------------------------ |
| **Runs in**            | Transaction     | Non-transactional context            |
| **Triggers realtime**  | Yes             | No (must call a mutation internally) |
| **Use for**            | CRUD operations | External API calls, long tasks       |
| **Optimistic updates** | Supported       | Not supported                        |

::note
Both composables require the realtime `ConvexClient`. They only work on the client side.
::

---
title: Pagination
description: Load data page by page with cursor-based pagination, infinite scroll patterns, and automatic resets on argument changes.
---

`useConvexPaginatedQuery` subscribes to a paginated Convex query and handles cursor tracking, page accumulation, and automatic resets when arguments change.

## Backend function

The Convex function must accept `paginationOpts` and return a `PaginationResult`.

```ts [convex/tasks.ts]
import { v } from 'convex/values'
import { query } from './_generated/server'

export const paginated = query({
  args: { status: v.optional(v.string()) },
  handler: async (ctx, args) => {
    let q = ctx.db.query('tasks').order('desc')
    if (args.status)
      q = q.filter(doc => doc.eq(doc.field('status'), args.status))
    return await q.paginate(args.paginationOpts)
  },
})
```

::note
You do not pass `paginationOpts` yourself. The composable injects it automatically with the cursor and page size.
::

## Basic usage

```vue [src/components/TaskFeed.vue]
<script setup lang="ts">
import { useConvexPaginatedQuery } from '@onmax/convex-vue'
import { api } from '../convex/_generated/api'

const { data, isPending, isDone, isLoadingMore, loadMore } = useConvexPaginatedQuery(
  api.tasks.paginated,
  {},
  { numItems: 20 },
)
</script>

<template>
  <p v-if="isPending">Loading...</p>
  <template v-else>
    <ul>
      <li v-for="task in data" :key="task._id">{{ task.text }}</li>
    </ul>
    <button v-if="!isDone" :disabled="isLoadingMore" @click="loadMore()">
      Load more
    </button>
  </template>
</template>
```

## Return values

::field-group
  ::field{name="data" type="ComputedRef<T[]>"}
  All loaded items flattened into a single array.
  ::

  ::field{name="pages" type="ComputedRef<T[][]>"}
  Items grouped by page. Useful when you need per-page boundaries.
  ::

  ::field{name="lastPage" type="ComputedRef<PaginationResult<T> | undefined>"}
  The raw result of the most recent page subscription.
  ::

  ::field{name="error" type="Ref<Error | null>"}
  The latest pagination error.
  ::

  ::field{name="isDone" type="Ref<boolean>"}
  `true` when the server reports no more pages.
  ::

  ::field{name="isPending" type="ComputedRef<boolean>"}
  `true` while the first page has not arrived yet.
  ::

  ::field{name="isLoadingMore" type="Ref<boolean>"}
  `true` while an additional page is loading.
  ::

  ::field{name="loadMore" type="() => void"}
  Request the next page. Does nothing when `isDone` is `true`.
  ::

  ::field{name="reset" type="() => void"}
  Clear all pages and reload from the first page.
  ::

  ::field{name="suspense" type="() => Promise<T[][]>"}
  Resolves when the first page arrives.
  ::
::

## Reactive arguments

When the arguments change, the composable resets automatically and reloads from the first page.

```vue [src/components/FilteredFeed.vue]
<script setup lang="ts">
import { useConvexPaginatedQuery } from '@onmax/convex-vue'
import { api } from '../convex/_generated/api'

const status = ref('active')

const { data, loadMore, isDone } = useConvexPaginatedQuery(
  api.tasks.paginated,
  computed(() => ({ status: status.value })),
  { numItems: 20 },
)
</script>
```

Changing `status` clears the loaded pages and starts a fresh subscription with the new filter.

::warning
Paginated queries use the realtime client only. SSR prefetching is not supported.
::

---
title: Controller
description: Connect, reconfigure, or disconnect the Convex client at runtime when the deployment URL is not available at startup.
---

The controller lets you manage the Convex connection lifecycle after the plugin has been installed. Use it when the deployment URL is not available during app startup.

## When to use

- Multi-tenant apps that choose a deployment at runtime.
- Embedded apps that receive configuration from a host container.
- Setups that wait for authentication before connecting.

::tip
Skip the controller when your deployment URL is available at startup. The default plugin setup is simpler.
::

## Install without a URL

Register the plugin with an empty options object. The plugin starts in a `disconnected` state.

```ts [src/main.ts]
import { convexVue } from '@onmax/convex-vue'
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)
app.use(convexVue, {})
app.mount('#app')
```

## Connect later

Access the controller through `useConvexController` and call `connect()` when the URL becomes available.

```vue [src/App.vue]
<script setup lang="ts">
import { useConvexController } from '@onmax/convex-vue/advanced'
import { onMounted } from 'vue'

const controller = useConvexController()

onMounted(() => {
  const url = window.localStorage.getItem('convex-url')
  if (url)
    controller.connect({ url })
})
</script>
```

`connect()` creates both the HTTP client and the realtime WebSocket client (in the browser). Composables like `useConvexQuery` start receiving data once the connection is established.

## Controller API

::field-group
  ::field{name="status" type="Ref<'connected' | 'disconnected'>"}
  The current connection state. Reactive — use it in templates or watchers.
  ::

  ::field{name="options" type="Ref<ResolvedConvexVueOptions>"}
  The current resolved plugin options. Read-only.
  ::

  ::field{name="connect" type="(options: { url: string, clientOptions?, server? }) => void"}
  Establish a new connection. Throws if no URL is provided.
  ::

  ::field{name="reconfigure" type="(options: Partial<ConnectOptions>) => void"}
  Update an existing connection with new options. Requires a previous `connect()` call.
  ::

  ::field{name="disconnect" type="() => void"}
  Close the WebSocket and clear both clients.
  ::

  ::field{name="getClient" type="() => ConvexClient | undefined"}
  Access the raw realtime client. Returns `undefined` when disconnected.
  ::

  ::field{name="getHttpClient" type="() => ConvexHttpClient | undefined"}
  Access the raw HTTP client. Returns `undefined` when disconnected.
  ::
::

## Reconfigure on tenant change

Use `reconfigure()` to switch between deployments without unmounting the app.

```vue [src/components/TenantSelector.vue]
<script setup lang="ts">
import { useConvexController } from '@onmax/convex-vue/advanced'

const controller = useConvexController()
const tenants = [
  { name: 'Acme Corp', url: 'https://acme.convex.cloud' },
  { name: 'Globex', url: 'https://globex.convex.cloud' },
]

function switchTenant(url: string) {
  controller.reconfigure({ url })
}
</script>

<template>
  <select @change="switchTenant(($event.target as HTMLSelectElement).value)">
    <option v-for="tenant in tenants" :key="tenant.url" :value="tenant.url">
      {{ tenant.name }}
    </option>
  </select>
  <p>Status: {{ controller.status.value }}</p>
</template>
```

## Create outside a component

`createConvexVueController` creates a controller without the Vue injection context. Use it when you need to manage the connection in a store or utility file.

```ts [src/convex.ts]
import { createConvexVueController } from '@onmax/convex-vue/advanced'

export const controller = createConvexVueController()
```

::warning
Queries, mutations, and actions throw if called before `connect()`. Check `controller.status` or guard your composable calls until the connection is established.
::

Nuxt Convex integrates [Convex](https://convex.dev) into your Nuxt application. The repo now ships two layers: a standalone Vue package and a Nuxt module built on top of it.

## Features

- **Real-time by default** — Queries auto-update when data changes
- **Shared Vue core** — `@onmax/convex-vue` powers both plain Vue and Nuxt apps
- **Virtual modules** — Import from `#convex` and `#convex/api`
- **Auto-imports** — Composables available without manual imports
- **File storage** — Upload API with auto-scaffolded Convex functions
- **DevTools** — Convex dashboard tab in Nuxt DevTools

## How It Works

Convex functions live in the `convex/` directory at your project root. When you write a query, mutation, or action, Convex generates TypeScript types automatically. `@onmax/convex-vue` provides the canonical Vue composables, and `nuxt-convex` layers Nuxt-specific aliases, auto-imports, renderless components, and storage scaffolding on top.

Your frontend subscribes to queries that update in real-time. When data changes on the server, all connected clients receive updates instantly without polling or manual refetching.

## Requirements

- Nuxt 3.0.0 or higher
- A Convex project (run `npx convex dev` to create one)

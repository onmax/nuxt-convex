---
title: Manual Initialization
description: Connect the Vue package later through the advanced controller entrypoint when the deployment URL only exists after app startup.
---

Install the root plugin first, then connect later through `@onmax/convex-vue/advanced` when the deployment URL only exists after app startup.

```ts [src/main.ts]
import { convexVue } from '@onmax/convex-vue'
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

app.use(convexVue, {})

app.mount('#app')
```

## Connect later

```vue [src/App.vue]
<script setup lang="ts">
import { useConvexController } from '@onmax/convex-vue/advanced'
import { onMounted } from 'vue'

const controller = useConvexController()

onMounted(() => {
  controller.connect({
    url: window.localStorage.getItem('convex-url') || 'https://your-app.convex.cloud',
  })
})
</script>
```

`connect()` creates the HTTP client and, in the browser, the realtime client. `reconfigure()` updates an existing connection and `disconnect()` clears it.

## When to use it

- Multi-tenant apps that choose a deployment at runtime.
- Setups that wait for host-provided configuration.
- Embedded apps that receive configuration from a host container.

## When not to use it

Skip the advanced controller when the deployment URL is already available during app startup. The default plugin setup is simpler and keeps the root API smaller.

---
title: Manual Initialization
description: Delay client creation in @onmax/convex-vue when the Convex deployment URL or auth state is only available later.
---

Set `manualInit: true` when you need to install the Vue plugin before you know which Convex deployment URL or auth configuration to use.

```ts [src/main.ts]
import { convexVue } from '@onmax/convex-vue'
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)

app.use(convexVue, {
  manualInit: true,
  url: 'https://placeholder.convex.cloud',
})

app.mount('#app')
```

## Initialize later

```vue [src/App.vue]
<script setup lang="ts">
import { useConvexContext } from '@onmax/convex-vue'

const { initClient } = useConvexContext()

onMounted(() => {
  initClient({
    url: window.localStorage.getItem('convex-url') || 'https://your-app.convex.cloud',
  })
})
</script>
```

`initClient()` recreates both the HTTP client and the realtime browser client. On the server, the browser client stays undefined by design.

## When to use it

- multi-tenant apps that choose a deployment at runtime
- setups that wait for an auth bootstrap step
- embedded apps that receive configuration from a host container

## When not to use it

Skip manual initialization when the deployment URL is already available during app startup. The default plugin setup is simpler and matches the Nuxt module behavior.

---
title: Module Configuration
description: Configure nuxt-convex with the real options exported from packages/nuxt/src/types/config.ts.
---

`nuxt-convex` exposes one module key: `convex`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-convex'],
  convex: {
    dir: 'convex',
    url: 'https://your-app.convex.cloud',
    server: true,
    storage: false,
    r2: false,
  },
})
```

## Options

| Option    | Type      | Default                                  | Meaning                                                                  |
| --------- | --------- | ---------------------------------------- | ------------------------------------------------------------------------ |
| `dir`     | `string`  | `'convex'`                               | Convex source directory. Relative paths are resolved across Nuxt layers. |
| `url`     | `string`  | `CONVEX_URL` or `NUXT_PUBLIC_CONVEX_URL` | Convex deployment URL.                                                   |
| `server`  | `boolean` | `true`                                   | Default SSR behavior for `useConvexQuery`.                               |
| `storage` | `boolean` | `false`                                  | Enables storage aliases and scaffolding.                                 |
| `r2`      | `boolean` | `false`                                  | Enables `#convex/r2` and `useConvexR2Upload`.                            |

## Environment variables

- `CONVEX_URL`
- `NUXT_PUBLIC_CONVEX_URL`

The module resolves `options.url` first, then `CONVEX_URL`, then `NUXT_PUBLIC_CONVEX_URL`.

## Layer resolution

The module searches layers for the Convex directory and generated API files. Reads prefer a layer that already contains generated files. Writes stay in the resolved directory when it is local and writable. If the resolved layer is read-only, scaffolding falls back to the project layer.

## Runtime config

The module writes these values to `runtimeConfig.public.convex`:

- `dir`
- `url`
- `r2`
- `storage`
- `server`

That runtime state is what the Nuxt plugin passes into the shared Vue context.

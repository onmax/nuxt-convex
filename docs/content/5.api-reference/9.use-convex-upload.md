---
title: useConvexUpload
description: Upload files through the storage feature entrypoint with a small client-side state wrapper around the generated upload URL flow.
---

`useConvexUpload` lives in `@onmax/convex-vue/storage` and wraps the generated upload URL flow exposed by `useConvexStorage`.

```vue [app/components/FileUpload.vue]
<script setup lang="ts">
import { useConvexUpload } from '@onmax/convex-vue/storage'

const { upload, isUploading, progress, error } = useConvexUpload({
  onSuccess: async (storageId, file) => {
    console.log(storageId, file.name)
  },
})
</script>
```

## Signature

```ts
function useConvexUpload(options?: {
  generateUploadUrl?: () => Promise<string>
  onSuccess?: (storageId: string, file: File) => void | Promise<void>
  onError?: (error: Error) => void
}): {
  upload: (file: File) => Promise<string | null>
  isUploading: DeepReadonly<Ref<boolean>>
  progress: DeepReadonly<Ref<number>>
  error: DeepReadonly<Ref<Error | null>>
}
```

## Important behavior

- Uploads are client-only.
- The helper sets `progress` to `0` before the request and `100` after a successful upload.
- The composable does not stream incremental browser upload progress.

::note
If you need byte-level progress callbacks, use the R2 path instead. [`useConvexR2Upload`](/api-reference/use-convex-r2-upload) tracks upload progress while it sends the file to the signed R2 URL and syncs metadata back through Convex.
::

---
title: useConvexUpload
description: Upload files through Convex storage with a small client-side state wrapper around the generated upload URL flow.
---

`useConvexUpload` wraps the generated upload URL flow exposed by `useConvexStorage`.

```vue [app/components/FileUpload.vue]
<script setup lang="ts">
const { upload, isUploading, progress, error } = useConvexUpload({
  onSuccess: async (storageId, file) => {
    console.log(storageId, file.name)
  },
})
</script>
```

## Signature

```ts
function useConvexUpload(options?: {
  generateUploadUrl?: () => Promise<string>
  onSuccess?: (storageId: string, file: File) => void | Promise<void>
  onError?: (error: Error) => void
}): {
  upload: (file: File) => Promise<string | null>
  isUploading: DeepReadonly<Ref<boolean>>
  progress: DeepReadonly<Ref<number>>
  error: DeepReadonly<Ref<Error | null>>
}
```

## Important behavior

- uploads are client-only
- the helper sets `progress` to `0` before the request and `100` after a successful upload
- it does not stream incremental browser upload progress

::note
If you need byte-level progress callbacks, use the R2 path instead. [`useConvexR2Upload`](/api-reference/use-convex-r2-upload) forwards progress data from `@convex-dev/r2/vue`.
::

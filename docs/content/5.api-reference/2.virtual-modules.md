---
title: Virtual Modules
description: Understand the Nuxt-only aliases created by nuxt-convex and when they become available.
---

`nuxt-convex` creates aliases so app code does not have to reach into generated or build directories directly.

## Always available

| Import        | What it exposes                                                       |
| ------------- | --------------------------------------------------------------------- |
| `#convex`     | Re-exports from `@onmax/convex-vue` through the Nuxt build directory. |
| `#convex/api` | Generated Convex API types from your resolved `convex.dir`.           |

```ts [app/pages/tasks.vue]
import { api } from '#convex/api'

const { data } = useConvexQuery(api.tasks.list, { userId: 'demo' })
```

## Available when `storage: true`

| Import                 | What it exposes                                                      |
| ---------------------- | -------------------------------------------------------------------- |
| `#convex/storage`      | `useConvexStorage` re-export through Nuxt.                           |
| `#convex/storage-refs` | Internal helper used by the Nuxt runtime to load storage references. |

## Available when `r2: true`

| Import       | What it exposes                      |
| ------------ | ------------------------------------ |
| `#convex/r2` | Re-export from `@convex-dev/r2/vue`. |

```ts [app/components/R2Upload.vue]
import { api } from '#convex/api'
import { useUploadFile } from '#convex/r2'

const uploadFile = useUploadFile(api.r2)
```

::note
Use the shared composables from `#convex` inside Nuxt. Import directly from `@onmax/convex-vue` only in standalone Vue apps.
::

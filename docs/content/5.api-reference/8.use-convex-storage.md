title: useConvexStorage
description: Use the storage feature entrypoint for upload URLs, reactive file URLs, and deletion.
---

`useConvexStorage` lives in `@onmax/convex-vue/storage`. Install the storage feature first, then use the helper for upload URLs, reactive file URLs, and deletion.

```ts [src/main.ts]
import { convexVueStorage } from '@onmax/convex-vue/storage'

app.use(convexVueStorage, {
  generateUploadUrl: api._hub.storage.generateUploadUrl,
  getUrl: api._hub.storage.getUrl,
  remove: api._hub.storage.remove,
})
```

```vue [app/components/FilePreview.vue]
<script setup lang="ts">
import { useConvexStorage } from '@onmax/convex-vue/storage'

const { generateUploadUrl, getUrl, remove } = useConvexStorage()

const url = getUrl('storage_id_here')
</script>
```

## Signature

```ts
function useConvexStorage(): {
  generateUploadUrl: () => Promise<string>
  getUrl: (storageId: string) => Ref<string | null>
  remove: (storageId: string) => Promise<void>
}
```

## Important behavior

- `generateUploadUrl()` throws when the storage feature is not configured or when the realtime client is disconnected.
- `getUrl()` returns a readonly ref and subscribes with `client.onUpdate(...)`.
- `remove()` calls the configured storage mutation through the current realtime client.

## When to use it

Use this helper when you want the storage primitives directly. Use [`useConvexUpload`](/api-reference/use-convex-upload) when you want the upload flow wrapped for you.

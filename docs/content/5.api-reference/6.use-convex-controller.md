---
title: useConvexController
description: Access the advanced controller entrypoint for delayed connection, reconfiguration, and raw client access.
---

`useConvexController` returns the advanced controller from `@onmax/convex-vue/advanced`. Use it when you need to connect later, reconfigure the deployment URL, disconnect, or access the underlying clients explicitly.

```vue [app/components/TaskInspector.vue]
<script setup lang="ts">
import { useConvexController } from '@onmax/convex-vue/advanced'
import { onMounted } from 'vue'

const controller = useConvexController()

onMounted(() => {
  controller.connect({
    url: window.localStorage.getItem('convex-url') || 'https://your-app.convex.cloud',
  })
})
</script>
```

## Behavior

- `connect()` requires a URL and creates the HTTP client plus the browser realtime client when running in the browser.
- `reconfigure()` updates the active connection without exposing internal refs.
- `disconnect()` clears the active connection.
- `getClient()` and `getHttpClient()` expose raw clients only through the advanced controller.

## Use it for

- delayed connection after app startup
- host-provided or multi-tenant runtime configuration
- controlled access to the raw Convex clients when the higher-level composables are not enough

Convex schemas define the database model the frontend helpers consume. Keep the schema in your Convex directory and generate the API after every change.

```ts [convex/schema.ts]
import { defineSchema, defineTable } from 'convex/server'
import { v } from 'convex/values'

export default defineSchema({
  tasks: defineTable({
    title: v.string(),
    userId: v.optional(v.string()),
    isCompleted: v.boolean(),
    createdAt: v.number(),
  }).index('by_user', ['userId']),
})
```

## Why indexes matter here

The query examples in this documentation use `.withIndex()`. Those examples only stay fast when the indexed fields match the filters you run from the frontend.

## Common validators

- `v.string()`
- `v.number()`
- `v.boolean()`
- `v.id('table')`
- `v.array(...)`
- `v.object(...)`
- `v.optional(...)`
- `v.union(...)`

## Storage-specific fields

If you enable scaffolded storage, your metadata tables typically reference `v.id('_storage')`.

```ts [convex/schema.ts]
uploads: defineTable({
  storageId: v.id('_storage'),
  name: v.string(),
  type: v.string(),
  createdAt: v.number(),
})
```
